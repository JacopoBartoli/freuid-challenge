Domain Adversarial Multiple Instance Learning for Document Fraud Detection

Jacopo Bartoli  ·  Jason Ravagli

July 2026 FREUID 0.068 Public Leaderboard MIT License

Abstract.

We present a domain-adversarial multiple instance learning (DA-MIL) system for detecting forged identity documents. Each document is resized to 672 × 896 and tiled into a 3×4 grid of 224×224 patches; each tile is passed through a frozen DINOv2 ViT-B/14 backbone, yielding 3 072 patch tokens per document. Gated attention pooling identifies the most discriminative tokens and produces a bag embedding. A gradient reversal layer suppresses template-specific cues during training, pushing the representation toward fraud signals that generalize across unseen document templates. On the FREUID Challenge 2026 public test set our best submission achieves a FREUID score of 0.068 under Protocol A.

Keywords — document fraud detection · multiple instance learning · domain adversarial training · vision transformer · DINOv2

1. Introduction

Detecting forged identity documents is challenging because fraud signals are localized visual artefacts — inconsistent fonts, digital re-printing traces, cloning marks — that vary in position, scale, and character across document templates. A model that learns template layout rather than fraud-specific texture generalizes poorly to the held-out templates used in competition evaluation.

Multiple instance learning (MIL) addresses the localization problem by treating each image as a bag of patch-level tokens and letting an attention mechanism identify which patches are discriminative, without any patch-level annotation. Domain adversarial training then prevents the bag aggregator from exploiting template-specific cues by reversing gradients flowing into a template classifier, so the shared representation is forced toward template-agnostic fraud features.

2. Method

2.1 Patch Feature Extraction with Tiling

Each document image is resized to 672 × 896 pixels using bicubic interpolation and then split into a 3×4 grid of non-overlapping 224×224 tiles. This preserves more of the original document resolution compared to a single 224×224 resize, retaining fine-grained forensic artefacts that would otherwise be lost.

Each tile is independently passed through a frozen DINOv2 ViT-B/14 backbone (vit_base_patch14_dinov2.lvd142m, pretrained on LVD-142M by Meta AI). With a patch stride of 14, each tile produces 256 spatial tokens of dimension d = 768. The 12 tiles per image are concatenated to form a bag of N = 3 072 tokens per document. The [CLS] token of each tile is discarded; only patch tokens are retained. The backbone weights are not updated during training.

2.2 Gated Attention MIL

Given a bag H = {h1, …, hN} ∈ ℝN×d, gated attention weights are computed as:

ak = softmaxk( wT [tanh(Vhk) ⊙ σ(Uhk)] )

The bag embedding z = Σk ak hk is a weighted sum that concentrates on the most discriminative patches. A linear classification head followed by a sigmoid produces the fraud probability. The MIL aggregator uses hidden dimension 256 and dropout 0.25.

2.3 Domain Adversarial Training

Template-specific biases are suppressed via a gradient reversal layer (GRL; Ganin et al. 2016) placed between the bag embedding z and a domain classification head. The GRL acts as the identity in the forward pass and negates gradients scaled by λ in the backward pass, causing z to become uninformative for template prediction while remaining useful for fraud detection. The total training loss is:

ℒ = ℒfraud + ℒdomain

To avoid destabilizing early training, λ follows a sigmoid warmup schedule:

λ(p) = λmax · ( 2 / (1 + e−γp) − 1 )

where p ∈ [0, 1] is fractional training progress, γ = 10, λmax = 1.0. At epoch 1 (≈ p = 0.04) λ ≈ 0.38; by epoch 25 it reaches 1.0. At inference time λ = 0: the domain head is inactive and the model runs as a standard attention MIL classifier.

3. Training Setup

We train under Protocol A: an 85 / 15 stratified split of the FREUID training corpus, stratified by document template and label. The validation split is used for early stopping; the best checkpoint by validation FREUID score is retained. For efficiency, DINOv2 features are extracted once and cached per sample to disk; the MIL head is the only module that receives gradient updates.

4. Inference

At inference time λ = 0 (domain head disabled). Each image is resized to 672 × 896 with bicubic interpolation and normalized with ImageNet statistics (μ = [0.485, 0.456, 0.406], σ = [0.229, 0.224, 0.225]). The image is tiled into a 3×4 grid of 224×224 patches; all 12 tiles are passed through the frozen DINOv2 backbone in a single batched forward pass, producing 3 072 patch tokens. The attention MIL head aggregates them and outputs a fraud probability in [0, 1] via sigmoid.

The Docker inference container reads all images under /data/ (recursively, by extension), scores them in batches of 32, and writes /submissions/submission.csv with columns id (filename stem) and label (fraud score). It runs fully offline (--network none): DINOv2 weights and the trained checkpoint are bundled in the image. The inference script is resume-friendly — if interrupted, it detects already-scored IDs and continues without re-scoring.

5. Results

The FREUID competition metric is defined as:

FREUID = 1 − 2 · gAuDET · gAPCER / (gAuDET + gAPCER)

where gAuDET is the genuine authentication detection rate and gAPCER = 1 − APCER is the attack detection rate. Lower is better; 0 is perfect.

The most significant improvement comes from tiling: preserving document resolution at 672×896 and splitting into 12 tiles (−0.127 FREUID vs. DA-MIL without tiling). With only 7 epochs of training the tiled model already outperforms all single-scale baselines, suggesting that the extra resolution is the dominant factor for this task. Domain adversarial training and token whitening both show diminishing returns relative to the resolution gain; whitening consistently hurt across all runs.

7. Reproduction

7.1 Requirements

  • Python 3.10+, PyTorch 2.x, CUDA-capable GPU (tested on GTX 1650 4 GB and RTX 3060)
  • Dependencies: timm, pandas, tqdm, Pillow, torchvision
  • Docker with --gpus all support (for the inference container)

7.2 Training from Scratch

train_online.py extracts DINOv2 features on-the-fly (no pre-caching required). It reads raw images directly and runs the full DA-MIL training loop end-to-end.

git clone <repository-url> cd freuid # Generate metadata.csv (maps image paths to labels and templates) python scripts/prepare_metadata.py --data-dir /path/to/freuid_data # Train DA-MIL — approximately 90 min on an RTX 3060 python scripts/train_online.py \ --data-dir /path/to/freuid_data \ --epochs 25 \ --lr 1e-4 # Best checkpoint saved to: # results/checkpoints/mil/a/da_full/fold0_best.pt

7.3 Docker Inference (Offline)

The Docker image bundles all weights and runs without network access. It maps /data/ → test images (read-only), /submissions/ → output directory.

# Build image (copies DINOv2 weights from HuggingFace cache) bash docker_build.sh # Run inference docker run --gpus all --network none \ -v /path/to/test/images:/data:ro \ -v /path/to/output:/submissions \ freuid-inference:latest # Output: /path/to/output/submission.csv (columns: id, label)
